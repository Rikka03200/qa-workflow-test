# 管理功能使用说明

## 新增功能概览

v3 新增两大管理功能，从顶部导航「**知识库**」和「**管理**」入口访问：

1. **CodeArts 容器路径管理**（/admin/modules）
2. **知识库管理**（/admin/kb）

---

## 1. CodeArts 容器路径管理

### 功能说明

可视化编辑 `_kb/projects/<product>/modules.md §2 CodeArts 用例存放目录树`。这个树定义了生成 `test-design.json` 时的容器节点路径（不等同于系统菜单，只是 CodeArts 中存放测试用例的文件夹）。

### 使用方式

1. **访问**：顶部导航「管理」→「CodeArts 容器路径」，或直接访问 `/admin/modules?product=wms`
2. **编辑树结构**：
   - 左侧编辑器：直接编辑节点文本、点击按钮增删改、上下移动
   - 右侧预览：实时显示 markdown 格式（保存后写入 modules.md）
3. **操作按钮**：
   - **添加根节点**：新增一级节点（如 web、POS、仓配app）
   - **+子**：为当前节点添加子节点
   - **删**：删除节点及其所有子节点（需确认）
   - **↑ ↓**：在同级节点中上下移动
4. **保存**：点击「保存」按钮，自动备份原文件（.bak-时间戳），写入新树结构

### 注意事项

- 保存前自动创建备份（`modules.md.bak-YYYYMMDD-HHMMSS`），出错可回退
- 树结构修改后，已生成的用例容器路径不会自动更新（需重新生成）
- 平台前缀（web、POS、仓配app 等）必须与 `modules.md §1` 保持一致（大小写敏感）

## 2. 知识库管理

### 功能说明

统一管理所有产品业务规则、术语、样本用例、全局规范。这是 AI 生成用例时的**唯一业务知识来源**。

### 使用方式

1. **访问**：顶部导航「知识库」，或直接访问 `/admin/kb?product=wms`
2. **文件分类**：
   - **🌐 全局规范**：跨产品通用（case-writing-spec.md、codearts-json-schema.md、qa-methodology.md）
   - **📦 产品文件**：当前产品（rules.md、terms.md、modules.md、样本等）
3. **查看/编辑**：
   - 左侧点击文件名 → 中间编辑器加载内容
   - 右侧自动生成章节导航（# ## ### 标题），点击跳转
   - 修改后点击「保存」，自动备份（.bak-时间戳）
4. **搜索**：
   - 点击编辑器右上角 🔍 图标，输入关键词
   - 实时显示匹配行 + 上下文（最多 50 条）
   - 点击结果直接跳转到对应文件和行号
5. **产品切换**：右上角下拉框切换产品线（WMS / 其他产品）

### 典型场景

#### 编辑业务规则（rules.md）
1. 左侧点击 `rules.md`
2. 右侧章节导航定位到目标模块（如 §53.2 分拣分组）
3. 编辑内容（保持章节号格式 `## 53.2 标题`）
4. 保存 → 自动备份

#### 添加新术语（terms.md）
1. 左侧点击 `terms.md`
2. 找到对应业务域章节（如 ## 拣货模块）
3. 按格式添加术语定义：
   ```markdown
   - **术语名**（英文/缩写）：定义。同义词：XXX。取值范围：AAA / BBB。
   ```
4. 保存

#### 搜索规则来源
1. 点击 🔍，输入关键词（如"分拣分组"）
2. 结果显示所有匹配的文件、行号、上下文
3. 点击结果跳转到具体位置查看完整章节

### 注意事项

- **保存前自动备份**：每次保存都创建带时间戳的 .bak 文件，可手动恢复
- **章节编号规则**：rules.md 的章节号（§1、§2.1 等）是手动维护的，编辑时注意保持连续
- **搜索范围**：搜索时会同时搜索全局规范 + 当前产品文件（不跨产品）
- **权限控制**：所有编辑直接写入 _kb/ 目录，请谨慎操作（有备份兜底）
- **格式要求**：
  - 章节标题：`## 1.2 标题`（必须有章节号，用于 AI 引用）
  - 规则来源标注：`[来源: EAR-xxx]` 或 `[来源: Confluence "页面名" §2]`
  - 术语定义：统一格式（粗体术语名 + 定义 + 同义词 + 取值）

## 功能架构

### 文件结构

```
_kb/
├── _global/                       # 全局规范（跨产品）
│   ├── case-writing-spec.md       # 测试点命名、步骤、预期规范
│   ├── codearts-json-schema.md    # JSON 契约、字段定义
│   └── qa-methodology.md          # 测试设计方法论
└── projects/
    └── <product>/                 # 产品级知识库（每产品一份）
        ├── README.md              # 产品概览
        ├── modules.md             # CodeArts 容器路径 + 系统菜单参考
        ├── rules.md               # 业务规则全集（按章节号组织）
        ├── terms.md               # 术语、缩写、同义词
        └── case-samples/          # 历史优质用例样本
```

### 技术实现

- **后端**：FastAPI `/admin/modules` `/admin/kb` 路由
- **前端**：原生 JS + Bootstrap 5（无额外依赖）
- **树编辑器**：DOM 操作 + contentEditable，实时预览 markdown
- **搜索**：Python 全文 grep（支持上下文显示）
- **备份机制**：每次保存前 `shutil.copy2` 创建 `.bak-时间戳` 文件

## FAQ

### Q1：误操作如何回退？
每次保存都自动创建备份（如 `rules.md.bak-20260610-153045`），找到最近的备份文件，重命名去掉 `.bak-时间戳` 后缀即可恢复。

### Q2：修改 modules.md 后已生成的用例会更新吗？
不会。容器路径是生成时写入 JSON 的，历史用例不受影响。如需统一调整，需要重新生成。

### Q3：知识库搜索为什么只返回 50 条？
防止结果过多影响性能。如需更精确搜索，可以缩小关键词范围（如加上模块名）。

### Q4：可以同时编辑多个文件吗？
当前版本一次只能编辑一个文件。如需批量修改，建议直接编辑 `_kb/` 目录下的 markdown 文件。

### Q5：rules.md 章节号如何维护？
手动维护。建议新增规则时：
1. 找到所属模块最后一个章节号
2. 新章节号 = 最后章节号 + 0.1（如 §53.2 后新增 §53.3）
3. AI 引用时必须标明章节号（如 `[来源: _kb/projects/wms/rules.md §53.2]`）
