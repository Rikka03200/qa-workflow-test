# WMS 模块地图

> 本文件同时维护三套路径概念，写用例时必须先区分：
> 1. **CodeArts 用例存放目录路径**：用于 `test-design.json` 的容器节点。
> 2. **系统菜单 / 操作路径**：用于 `condition` / `step` / `expect` 的执行定位。
> 3. **测试点标题模块路径**：用于 `{平台}_{模块路径}_{测试要点}` 的标题命名。
>
> 详细业务规则到 `rules.md` 对应章节查（每个模块都有 "→ rules.md §X" 提示）。

---

## 1. 平台一览

| 平台 | 标识（命名前缀，大小写敏感）| 范围 |
|---|---|---|
| WMS Web | `web` | PC 端管理后台 |
| 仓配 App | `仓配app` | 移动端：拣货 / 收货 / 复核 / 上架 / 分拨 / 补货 / 移库 / 库存查询 / 销退收货 / 容器拆并 / 盘点 |
| 采配 App | `采配app` | 移动端：直配采购录入 / 采购任务 / 要货查询 / 收货 / 装车 / 复核 |
| 零售 App | `零售app` | 乐檬零售；与 WMS 直配模块对接 |
| POS | `POS`（**大写**）| POS 端：拣货任务（生鲜）/ 收货称重打印 / 分拣任务 / 系统设置 等 |
| TMS 司机小程序 | `TMS`（**大写**，容器节点也可写 `TMS小程序`）| 物流端：运输任务 / 装车任务 / 司机交接 |
| 供应商 App | `供应商app` | 供应商端，与"供应商平台"对接 |
| 前置仓 App | `前置仓app` | 前置仓配套 |
| PDA | `PDA` | 老 PDA 端（部分场景）|

> **大小写硬规则**：`POS` / `TMS` / `PDA` 全大写；`web` / `仓配app` / `采配app` / `零售app` / `供应商app` / `前置仓app` 全小写。违反即视为命名错误。

## 2. CodeArts 用例存放目录树（JSON 容器路径真源）

`test-design.json` 中测试点前面的所有普通容器节点，必须来自本节目录树。它只是 CodeArts 中存放测试用例的文件夹路径，不等同于系统菜单或功能模块位置。

**硬规则**：

- JSON 容器路径 = 工单号 → CodeArts 用例存放目录 → 测试点节点。
- 不要把系统模块、功能切面、规则名、按钮名自创成 JSON 容器节点。
- 系统菜单 / 操作位置只写进 `step.text`，不要反推为 JSON 容器路径。
- 测试点标题仍按 `_kb/_global/case-writing-spec.md §1` 使用 `{平台}_{模块路径}_{测试要点}`，不要因为容器路径变化而改成文件夹名堆叠。
- 例如 `EAR-244733` 的容器路径应归到：
  - `EAR-244733 > web > 设置管理 > 基础设置 > 系统参数 > [测试点]`
  - `EAR-244733 > web > 设置管理 > 基础设置 > 配送客户 > [测试点]`

### 3.3 系统菜单层级写法（步骤定位参考）

`step.text` 中的页面路径按系统实际菜单或页面入口写清楚；tab/section 可写进步骤动作或测试点标题，但不要反推为 JSON 容器节点。

✅ JSON 容器路径：

```text
EAR-xxx > web > 标品管理 > 基础设置 > 标准参数 > [测试点]
```

✅ 步骤定位：

```text
wms-标品管理-基础设置-标准参数-上架参数，切换参数“X”=true
```

❌ 错误：

```text
EAR-xxx > web > 标品管理 > 基础设置 > 标准参数 > 上架参数 > [测试点]   ← 把 tab/section 当成 JSON 容器层
EAR-xxx > web > 标准管理 > 标准参数 > 上架参数 > [测试点]              ← 自创不存在的 CodeArts 目录层
```

### 3.4 公共页 / 公共组件 不建独立目录（硬规则）

"选择库位"公共页、"WMS 商品"公共页、"配送客户"公共页、"打印格式"公共页等**不是独立菜单**——它们是被嵌入到具体业务流程的步骤里调用的。

❌ 错误（把公共页当 web 一级模块平级处理）：

```
EAR-xxx > web > "选择库位"公共页 > [测试点]
```

✅ 正确（公共页是某流程的 step，写到所属流程的 step 中）：

```
EAR-xxx > 仓配app > 上架 > [测试点: 仓配app_上架_..._仅显示分拣位]
  └─ step: 仓配app-上架，选择商品A，点击"上架库位"
  └─ expect: 选择库位公共页仅显示分拣位...
```

> 注：rules.md §246 把“选择库位公共页”列为独立章节是为了**业务规则聚合**（公共页本身的规则集中沉淀），**与 JSON 容器路径无关**。写用例时容器路径以 `modules.md §2` 的 CodeArts 用例存放目录树为准。
