# WMS 业务术语表

> AI 遇到陌生术语先查这里；若缺失，标 `[待确认: 术语 X 含义]`，**不要凭通用常识猜测**。
> 本表只放 WMS+仓配 App 域术语；跨产品通用术语放 `_kb/_global/`。

---

## 仓库基础

| 术语 | 定义 | 备注 |
|---|---|---|
| 仓库 | 物理仓储单位，顶层 | 一个系统可有多仓库 |
| 库区 | 仓库下的逻辑分区 | 一仓多库区 |
| 库位 | 库区下的最小存储单位 | 有类型（分拣位/其他）|
| 分拣位 | 库位类型之一 | "低优先级"参数仅此类型可勾选 |

## 库存形态

| 术语 | 定义 |
|---|---|
| 整件 | 完整包装单位（如 1 箱）|
| 拆零 | 拆开包装后的散件（如 1 包）|
| 折算系数 | 1 整件 = N 拆零（如 1 箱=10 包，按商品配置）|
| 系统转化 | 拆零需求 ≥ 折算系数时，自动转为"整件+剩余拆零" |

## 商品与批次

| 术语 | 定义 |
|---|---|
| 商品 | 档案单元，含 WMS 商品标准属性 |
| 批次 | 同商品同生产日期的库存集合 |
| 生产日期 | 批次属性，用于先进先出/后进先出 |
| 批次可拣量 | 当前批次在某库位上可拣货的数量 |

## 订单与拣货

| 术语 | 定义 |
|---|---|
| 出库订单 | 客户出库需求单，含多商品 |
| 波次策略 | 生成波次的规则配置 |
| 波次计划单 | 按波次策略生成，含多个出库订单的拣货任务 |
| 拣货任务 | 波次计划单内最小拣货单元 |
| 拣货员 | 在仓配 App 执行拣货的用户 |
| 领取 | 拣货员从任务池领取拣货任务的动作 |
| 取消领取 | 拣货员/管理员撤销已领取状态 |
| 一键拣货完成 | 仓配 App 中的快捷完成功能（不应用批次推荐规则）|

## 分拣

| 术语 | 定义 |
|---|---|
| 分拣方式 | 波次策略字段，含"按单分拣"等 |
| 按单分拣 | 按订单维度拣货 |
| 分拣分组（按库位）| 组内挂库位 + 拣货员 |
| 分拣分组（按商品）| 组内挂商品 |
| 跨分组领取 | 拣完本组再领其他组（受"允许跨分组"参数控制）|

## 锁库与不锁库

| 术语 | 定义 |
|---|---|
| 锁库流程 | 拣货时库存被锁定的流程（默认）|
| 不锁库流程 | 不锁定库存的流程；批次推荐规则**仅应用此流程**且仅按单分拣 |

## 推荐规则相关（2026 新增）

| 术语 | 定义 |
|---|---|
| 批次推荐规则 | 标准管理-业务规则下的新模块，含 4 个优先级维度 |
| 推荐值 | 批次推荐规则内的 4 类优先级：库存形态、生产日期、库位库存、分拣分组 |
| 低优先级（库位）| 库位参数；勾选后批次推荐优先级最低，仅不锁库按单分拣应用 |
| 推荐库位（一品多位）| 波次策略中的旧规则，同商品多个库位时的选择规则 |

## 参数（标准管理-拣货参数）

| 参数 | 选项 / 默认 | 行为 |
|---|---|---|
| 仅允许领取生产日期最早商品 | true / false | 开启后不应用批次推荐规则 |
| 允许跨分组领取拣货任务 | 按分组 / 按波次按分组 / 按客户 | 开启（任一非空选项）后不应用批次推荐规则 |
| 允许跨波次领取任务 | true / false | 开启后不应用批次推荐规则 |

## UI / 平台

| 术语 | 定义 |
|---|---|

## 采配 App 域（2026 大宗回溯沉淀）

| 术语 | 定义 |
|---|---|
| 采配 App | 移动端采购+配货应用（区别于"仓配 App"——拣货应用）|
| 直配商品 | 非标 WMS 无仓流程下，门店下单汇总后由市场采购直接配送的商品 |
| 非标 WMS 无仓流程 | 客户使用 WMS 但不走仓库存储、直接由市场采购配送的业务模式 |
| 多采 | 实际采购量 > 门店原订货量的部分 |
| 强配 | 把多采货品按规则分配给门店 |
| 订货比例分配 | 按"门店要货 / 订货总量"比例把采购量分摊到各门店的算法（见 rules.md §8.2）|
| 批次换算率 | 1 整件包含的拆零数（如 1 件 = N 公斤）|
| 未配基数 | 订货总量 − 总已配量，多批次场景下的分母 |

## 集货与托盘（2026 累积，参考 rules.md §10.2 / §11.5）

| 术语 | 定义 |
|---|---|
| 集货位 | 仓库分拣完成后临时存放客户货品的库位（一个集货位放一个或多个客户）|
| 集货托盘 | 拣货员集货时使用的托盘，可选"空 / 自动生成托盘号 / 扫描托盘号 / 预生成托盘号"|
| 预生成托盘号 | 集货前批量生成、打印的托盘号（YRQ 开头、唯一、30 天内有效）|
| 临时集货位 | 拣货过程中临时使用的集货位，不受"同波次同客户仅集货同集货位"约束 |
| 月台 | 装车口，关联集货位；调度管理排车时指定 |
| 子集货位 | 集货位下编号 1~100 的细分（业务约定，未额外建模）|
| 集货位剩余体积 | 库位承载体积 − 库内库存商品体积 − 当天已分配未集货的拣货任务商品体积 |

## DPS 分拣（2026 累积，参考 rules.md §10.6/§10.7）

| 术语 | 定义 |
|---|---|
| DPS 分拣 | 电子标签辅助分拣，对应 PDA 分拣的另一种方式 |
| 仅总提 | WMS 波次策略分拣方式之一，先总提后分拨（仅总提 + DPS 是 2026 新组合）|
| DPS 库位 | DPS 分拣对应的库位类型，仅总提领取任务不匹配 DPS 库位库存 |
| WMS 商品-DPS 分拣标记 | 是否拆零=是 时激活默认 true；是否拆零=否 置灰默认 false |

## 配货任务（生鲜，参考 rules.md §12 / §13）

| 术语 | 定义 |
|---|---|
| 配货任务 | WMS 生鲜管理-配货任务，向客户分配可配数量 |
| 等比例分配（规则二）| 配送数量 = 按订货数量等比例分配（详见 rules.md §13）|
| 批次+辅助商品 | 批次商品启用辅助单位；对 WMS 即批次商品，配送单位=辅助单位 |
| 强配任务 | 配货任务的特殊类型，独立模板导入，forceAllocate=true |
| 容器商品绑定 | WMS 生鲜管理-容器商品绑定，一商品仅允许绑定一容器 |
| 容器数量 | 配货数量对应基本单位数量 / 容器商品绑定装载量，舍入 |

## 业务模式（WMS 三大形态，rules.md §70 §84 §111 §139 §189 §45 §100 §186）

| 模式 | 定义 | 主要差异 |
|---|---|---|
| 标品 WMS | 标准仓 WMS，最完整 | 出库订单 + 波次 + 拣货 + 复核 + 集货 全流程 |
| 生鲜 WMS | 生鲜管理，强配 + 等比例分配 | 多按"配货任务"分配，时效要求高 |
| 非标 WMS / 采配 WMS | 不走仓库存储 | 门店下单汇总 → 市场直接采购 → 配送（"采配 App-直配商品"专题）|
| 前置仓 WMS | 简化版 WMS（2026 新增）| 模块裁剪，靠近门店的小仓库；与"前置仓 App"配套 |

> 同客户可同时启用多种模式。写用例时区分清楚目标模式：标品业务 ≠ 生鲜业务 ≠ 非标业务。

## 仓内动作区分（易混淆，rules.md §29 §35 §37 §38 §178 §17 §245 §22 §6）

| 动作 | 定义 | 触发场景 |
|---|---|---|
| 收货 | 供应商送到 → 库存入账 | 仓配 App-收货 / 收货任务 / 入库订单 |
| 上架 | 收货后把货物搬到库位（指定库位）| 仓配 App-上架 |
| 移库 | 库位之间转移库存 | 仓配 App-移库 / 移库任务 |
| 补货 | 拣货区库存不足 → 从备货区补入 | 仓配 App-补货 / 超量补货 |
| 分拨 | 总提 + 后续按门店分拨拣货 | 仓配 App-分拨 / "总提 + 分拨"流程 |
| 拣货 | 按订单/波次从库位取出货物 | 仓配 App-拣货（锁库/不锁库）/ PDA / DPS |
| 分拣（Pos）| 集中拣货完成后按门店分拣 | 分拣 Pos / DPS / 输送线 |
| 总提 | 多客户合并一次性拣货后再分拨 | "仅总提" 波次策略 |
| 集货 | 拣货完成后按客户/门店暂存到集货位 | 仓配 App-集货 / 集货位 / 集货托盘 |
| 复核 | 集货后扫码核对数量与目的地 | 仓配 App-复核 / 集货复核 |
| 装车 / 发车 | 集货 → 装车 → 司机离场 | 装车单 / 装车任务 / 发车任务 / 发车单 |
| 盘点 | 周期性核对账面库存 vs 实物 | 仓配 App-盘点 / 盘点任务 / 盘点单 / 盘点计划 |

> **拣货 vs 分拣**：拣货是"从库位取出"，分拣是"取出后按门店分组"；两者可同岗位但行为不同。
> **总提 vs 按单**：总提是先合并采集再拆分配送，按单是边采集边分配。

## WCS / 设备 / 平台辨析（rules.md §61 §103 §82 §188 §77 §154 §191 §202 §205）

| 名词 | 定义 |
|---|---|
| WCS | Warehouse Control System，控制系统，对接快仓 2.0 / 宝开 2.0 等智能仓硬件 |
| RCS | 机器人调度系统，部分客户使用，承担集货 / 移库 |
| DPS | Digital Picking System，电子标签辅助分拣（与 PDA 分拣并列的另一种方式）|
| 提升机 | 智能仓垂直运输设备（仅 4301 博全场景）|
| 输送线 | 整件分拣线体，配套"输送线分拣条码" |
| 格口 | 输送线 / 分拣线末端的目的格子，对应门店 / 月台 |
| 智能仓 | 包含 WCS + 提升机 + 输送线 + 输送线分拣 + 自动分拨墙 等设备组合 |
| 月台 | 装卸货操作平台（与"格口"关联），有"月台调度"和"月台管理"两模块 |
| 集货位 vs 月台 | 集货位是仓内临时库位；月台是仓门口装车口，可关联多个集货位 |

## 客户账套与定制（rules.md §0.16 速查）

- 账套号是客户唯一标识（如 4301 = 博全；4173 = 戴永红；29233 = 美多）
- **客户定制规则不可作通用规则使用**——若工单不属于该客户，相关章节内容禁止援引
- 写测试用例时若涉及定制章节，在 `condition` 中**必须显式标注账套**
