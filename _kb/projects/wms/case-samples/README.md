# WMS 历史用例样本

## `historical-cases.json.md`

来源：迁移自 `D:\Downloads\测试用例\之前的一些用例(不一定对).md`。

- 内容：仓配 App 拣货优先级（多客户单库位 / 多库位）系列的 CodeArts 测试设计 JSON
- 体量：~88KB（一行 JSON）
- 用途：**作为风格样本**——AI 写新用例时参考标题命名、前置条件结构、步骤+预期分层
- **注意**：文件名带"不一定对"——AI 引用其中具体业务规则时**要谨慎**，优先以 `rules.md` 为准

### 风格学习点

- 测试点标题：`仓配app_拣货_领取优先级_<策略>_<客户场景>_<参数取值>`
- 前置条件用编号 1./2./3./4./5.（参数、库位、分组、波次策略、订单），`<br>` 换行；关键值用中文双引号或上下文表达，禁止 `<b>` 加粗
- 步骤动词化、含具体定位：`登录用户B仓配App-拣货，点击领取`
- 每个领取动作后跟一个"取消领取"复位步骤，方便后续步骤独立验证
- 预期格式：`领取成功，拣货录入显示分拣库位=X-XX，生产日期=YYYY-MM-DD，批次可拣量=N公斤`

### 已知问题（参考时注意）

历史样本中有损坏 HTML：`《span style="font-weight: bold;"span》...《/spanspan》`。
- 这是从富文本编辑器复制时的转义破损
- AI **不要**复用这种格式
- 禁止输出 `<b>...</b>`；需要强调时用中文双引号或让字段和值本身表达清楚
